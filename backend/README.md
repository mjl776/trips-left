## Overview

This is the backend for the fantasy football optimizer, built with [NestJS](https://nestjs.com) and [Prisma](https://www.prisma.io) (PostgreSQL/Supabase). It owns all the actual logic — league/roster data, fantasy scoring, and player analytics — so the frontend stays a thin client that just renders whatever this API returns.

Real historical NFL stats are ingested separately via Python scripts in `ingestions/` (pulling from [nflverse](https://github.com/nflverse)), not generated by this app.

## Modules

- **`league`** — create/import leagues and their scoring settings (`create-mock-league`, `import-sleeper-league/:leagueId`, `view-sleeper-league/:leagueId`).
- **`lineup`** — roster construction: create a lineup and assign/swap/drop players into slots (`create-lineup`, `add-player`, `add-drop-player`, `swap-players`, `view-lineup`).
- **`player`** — player lookups and season-level analytics (`players`, `view-player`, `player-stat-rank`).
- **`projections`** — fantasy scoring math: forward projections (`projections`) and per-roster insights — best/worst player and a dark-horse pick based on advanced metrics (`lineup-insights`).

## Running locally

```bash
npm install
npm run start:dev
```

Runs on port `8080` by default (override with `PORT`). Needs `DATABASE_URL` (pooled, for the app) and `DIRECT_URL` (unpooled, for Prisma migrations/scripts) set in `.env`.
